<p> <br>Projeto acadêmico de uma plataforma completa e fullstack de uma instituição fictícia que oferece serviços para agendamento de consultas 
  médicas online com integração de previsão do tempo (OpenWeather) e consulta de CEP (ViaCEP). Inclui sistema robusto de autenticação e design responsivo.
Desenvolvida para uma avaliação da disciplina Desenvolvimento Backend I, no curso de Análise e Desenvolvimento de Sistemas da Universidade Veiga de Almeida.</p>

<br>

<h2>Tecnologias:</h2>
<ul>
  <li> <b>Backend:</b> Node.js + Express + SQLite (clinica.db) + JWT Authentication + CORS configurado</li>
  <li> <b>Frontend:</b> Vue.js 3 + Vite + Tailwind CSS + Axios (HTTP) + Vue Router</li>
  <li> <b>APIs externas:</b> OpenWeather API (previsão por coordenadas) + ViaCEP (consulta CEP Brasil)</li>
  <li> <b>Design:</b> Adobe Photoshop + Adobe Illustrator + Figma</li>
  <li> <b>Hospedagem:</b> Vercel (Frontend) + Render (Backend)</li>
</ul>

<br>

<h2>Funcionalidades:</h2>
<ul>
  <li> Autenticação JWT (Login + Register completos); </li>
  <li> CRUD agendamentos via REST API; </li>
  <li> Previsão climática automática (OpenWeather API); </li>
  <li> Consulta CEP inteligente (ViaCEP API); </li>
  <li> Dashboard responsivo consultas; </li>
  <li> Validação formulários frontend/backend; </li>
  <li> Comunicação Frontend-Backend segura (CORS); </li>
  <li> SQLite persistência (User + Appointment); </li>
  <li> Deploy produção Vercel + Render; </li>
  <li> .env protegido (Gitignore + Environment Variables); </li>
  <li> Design responsivo mobile-first; </li>
  <li> Loading states e error handling; </li>
  <li> Token validation rotas protegidas. </li>
</ul>

<br>

<h2>Aprendizados:</h2>
<ul>
  <li> Gestão Git avançada (monorepo, branch, PR, gitignore); </li>
  <li> Environment Variables (.env local + Vercel/Render); </li>
  <li> Autenticação JWT completa (login/register/refresh); </li>
  <li> SQLite + Express ORM (models User/Appointment); </li>
  <li> Vue 3 Composition API + Vite; </li>
  <li> RESTful API design (/api/auth, /api/appointments); </li>
  <li> Deploy contínuo (Vercel auto-deploy + Render); </li>
  <li> Tailwind CSS responsivo mobile-first; </li>
  <li> Error handling estruturado (frontend/backend); </li>
  <li> Segurança API keys (nunca hardcoded); </li>
  <li> Debug cross-origin (localhost ➔ production); </li>
  <li> Monorepo estrutura (backend + frontend). </li>
  
</ul>
